# AI Readiness Workshop
## Facilitator Guide

---

## Overview

**Purpose**: Assess whether a client organisation is ready to adopt AI automation, identify gaps, and create a readiness roadmap.

**Duration**: Half-day (3.5-4 hours)

**Outcome**: Readiness scorecard across 4 dimensions, gap analysis, and recommended actions to improve readiness.

**Ideal Participants**: 5-8 people including IT, operations, data/analytics, HR/change management, and executive sponsor.

**When to Use**: 
- Before committing to an AI project
- When a client is interested but unsure if they're ready
- After an Opportunity Assessment, to validate readiness for the top opportunity

## Section-by-Section Facilitation Notes

### 1. Welcome & Context (15 min)

**Slide**: Title + Why Readiness Matters

**Facilitator Script**:
> "Thanks for joining today. Before any organisation invests in AI automation, it's worth asking: are we actually ready for this?
>
> AI projects fail not because the technology doesn't work, but because the organisation wasn't prepared — bad data, systems that don't talk to each other, teams that aren't ready for change.
>
> Today we're going to honestly assess where you stand across four dimensions, identify gaps, and create a practical plan to address them."

**Key Points**:
- Readiness ≠ perfection. No one is 100% ready.
- The goal is to know what you're working with and what needs attention.
- Better to find gaps now than discover them mid-project.

**Activity**: Quick check-in — "On a scale of 1-10, how ready do you think your organisation is for AI? Write it down. We'll revisit at the end."

---

### 2. Readiness Framework (20 min)

**Slide**: The Four Dimensions

**Present the framework**:

| Dimension | What We're Assessing |
|-----------|---------------------|
| **Data Readiness** | Is your data accessible, accurate, and governed? |
| **Technical Readiness** | Can your systems support AI integration? |
| **Organisational Readiness** | Do you have the skills, sponsorship, and change capacity? |
| **Process Readiness** | Are your processes documented and standardised? |

**Scoring Scale** (use consistently throughout):
- **1 - Not Ready**: Significant gaps, would block or derail a project
- **2 - Early Stage**: Some foundations, but major work needed
- **3 - Developing**: Making progress, gaps are addressable
- **4 - Capable**: Solid foundation, minor improvements needed
- **5 - Advanced**: Strong position, ready to go

**Facilitator Tips**:
- Emphasise honesty over optimism
- "We're not here to look good, we're here to get clarity"
- Each dimension will be scored separately, then combined

---

### 3. Data Readiness (45 min)

**Slide**: Data Readiness Assessment

**Key Questions to Work Through**:

**A. Data Availability (10 min)**
- Where does the data for this process live?
- Can we access it programmatically (APIs, database, exports)?
- Is there a single source of truth, or multiple conflicting sources?
- How much historical data is available?

**B. Data Quality (15 min)**
- How accurate is the data? (Ask for examples of known issues)
- How complete is it? (Missing fields, gaps)
- How consistent is it? (Same thing recorded different ways)
- How timely is it? (Real-time, daily, weekly?)

**C. Data Governance (10 min)**
- Who owns the data?
- Are there data dictionaries or documentation?
- What are the privacy/security classifications?
- Are there restrictions on how data can be used?

**D. Data Integration (10 min)**
- How is data currently shared between systems?
- Are there existing ETL processes or data pipelines?
- What format is the data in?

**Scoring Exercise**:
Have participants score each sub-area in their workbooks, then discuss and agree on a consensus score for Data Readiness overall.

**Red Flags to Listen For**:
- "It depends who you ask"
- "We have data quality issues but we work around them"
- "That's in a spreadsheet on someone's desktop"
- "We'd need IT to extract that"

---

### 4. Break (15 min)

**Facilitator Action**:
- Capture the Data Readiness score
- Note any specific gaps mentioned
- Prepare for Technical Readiness section

---

### 5. Technical Readiness (40 min)

**Slide**: Technical Readiness Assessment

**Key Questions to Work Through**:

**A. System Architecture (10 min)**
- What are the core systems involved in this process?
- How old are they? (Legacy vs modern)
- Is there documentation of the architecture?

**B. Integration Capability (15 min)**
- Do systems have APIs? Are they documented?
- What integration patterns exist today? (Point-to-point, middleware, manual)
- Are there existing integration platforms (MuleSoft, Boomi, etc.)?
- What's the typical timeline to build a new integration?

**C. Infrastructure (10 min)**
- Where would AI workloads run? (Cloud, on-prem, hybrid)
- Is there capacity for additional compute?
- What are the security/network constraints?

**D. IT Capacity (5 min)**
- Does IT have bandwidth to support an AI project?
- What's the typical lead time for IT requests?
- Is there a backlog that would delay this?

**Scoring Exercise**:
Score each sub-area, agree on consensus Technical Readiness score.

**Red Flags to Listen For**:
- "That system doesn't have an API"
- "We'd need to go through a 6-month security review"
- "IT is fully allocated for the next year"
- "We're in the middle of a system migration"

---

### 6. Organisational Readiness (40 min)

**Slide**: Organisational Readiness Assessment

**Key Questions to Work Through**:

**A. Executive Sponsorship (10 min)**
- Is there a clear executive sponsor for AI initiatives?
- Do they have budget authority?
- How engaged are they? (Championing vs passive support)
- Is AI on the strategic agenda?

**B. Skills & Capabilities (10 min)**
- Does anyone in-house understand AI/ML?
- What's the technical sophistication of the team who'd use this?
- Is there a data science or analytics function?
- What training would be needed?

**C. Change Capacity (10 min)**
- How does the organisation typically handle change?
- What other major initiatives are in flight?
- Is there change fatigue?
- What's the track record of technology adoption?

**D. Culture & Appetite (10 min)**
- How do people feel about AI? (Excited, fearful, sceptical)
- Is there trust in automation?
- Are people protective of their current roles?
- What's the communication like about AI?

**Scoring Exercise**:
Score each sub-area, agree on consensus Organisational Readiness score.

**Red Flags to Listen For**:
- "Leadership says they want AI but won't commit budget"
- "We just went through a big restructure"
- "People are worried about their jobs"
- "We've tried automation before and it didn't go well"

---

### 7. Break (15 min)

**Facilitator Action**:
- Capture Organisational Readiness score
- Start thinking about patterns and themes
- Prepare Process Readiness section

---

### 8. Process Readiness (30 min)

**Slide**: Process Readiness Assessment

**Key Questions to Work Through**:

**A. Documentation (10 min)**
- Is the process documented?
- When was documentation last updated?
- Does documentation reflect reality?
- Are decision rules and exceptions captured?

**B. Standardisation (10 min)**
- Is the process performed consistently?
- How much variation is there between people/teams?
- Are there workarounds and unofficial processes?
- What percentage is "standard" vs "exception"?

**C. Ownership (5 min)**
- Who owns the process?
- Is there clear accountability?
- Who can make decisions about changing it?

**D. Measurement (5 min)**
- Is the process measured today?
- What metrics exist?
- Is there baseline data to compare against?

**Scoring Exercise**:
Score each sub-area, agree on consensus Process Readiness score.

**Red Flags to Listen For**:
- "The documentation is out of date"
- "Everyone does it slightly differently"
- "It's not really anyone's process"
- "We don't really track that"

---

### 9. Scoring & Gap Analysis (20 min)

**Slide**: Readiness Scorecard

**Activity**: Consolidate all scores into the summary scorecard

| Dimension | Score (1-5) | Key Gaps |
|-----------|-------------|----------|
| Data Readiness | | |
| Technical Readiness | | |
| Organisational Readiness | | |
| Process Readiness | | |
| **Overall** | (average) | |

**Interpretation Guide**:
- **4-5 Average**: Ready to proceed with confidence
- **3-4 Average**: Ready with some preparatory work
- **2-3 Average**: Significant gaps to address first
- **1-2 Average**: Not ready — foundational work needed

**Gap Prioritisation Exercise** (10 min):
1. List all identified gaps on sticky notes
2. Plot on a 2x2 matrix: Impact vs Effort to Fix
3. Identify the critical gaps (high impact, blocking)

**Discussion**:
- Which gaps would block an AI project?
- Which gaps are quick wins?
- Which gaps need longer-term investment?

---

### 10. Roadmap & Close (15 min)

**Slide**: Readiness Roadmap

**Activity**: Create a simple action plan

| Gap | Action | Owner | Timeline | Priority |
|-----|--------|-------|----------|----------|
| | | | | |

**Typical Actions by Dimension**:

**Data**:
- Data quality cleanup project
- Implement data governance
- Build data dictionary
- Create API access

**Technical**:
- API development for key systems
- Cloud infrastructure setup
- Security review process
- Integration platform evaluation

**Organisational**:
- Executive alignment session
- AI awareness training
- Change management planning
- Skills assessment

**Process**:
- Process documentation project
- Standardisation initiative
- Process ownership assignment
- Baseline measurement setup

**Wrap-Up Script**:
> "Let's revisit your initial readiness score. Has your view changed?
>
> The goal today wasn't to find out you're perfect — it was to get clarity on where you stand and what needs attention.
>
> We'll send you a full report with the scores, gaps, and recommended actions within a week."

**Capture**:
- Final scores
- Prioritised gaps
- Agreed actions
- Any open questions
